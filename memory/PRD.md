# KitchenNotes — PRD

## Vision
A tiny tool for Indian cloud kitchen owners: tap a customer name + order number → get a warm, Hinglish, hand-written sticky note generated by Gemini, then print it on a 58mm Bluetooth thermal printer so the customer smiles, re-orders, and tells their friends.

## Users
Cloud kitchen operators (single-operator brands and small teams).

## Core features (v1)
1. **First-launch setup** — store Gemini API key + kitchen/brand name on-device (AsyncStorage). Both required.
2. **Bluetooth printer setup** — scan, list, connect to a 58mm ESC/POS Bluetooth printer. Shown automatically every launch if no printer is connected. Skippable.
3. **Main input** — Customer Name + Order Number (defaults to 1, has a stepper). Order number persists across sessions and auto-increments after each successful print.
4. **Quote generation** — `POST /api/generate-quote` calls Gemini 2.5 Flash (via `emergentintegrations`) with the user's own key. Tone is randomly picked from `warm | funny | motivational`. Output: 2-line Hinglish note ending with an em-dash kitchen sign-off.
5. **Preview / Sticky Note screen** — generated quote rendered on a mustard sticky note with a 2-degree rotation, masking-tape detail, and a handwritten font (Caveat). `Regenerate` re-calls Gemini with the same inputs. `Print to 58mm` formats a 32-char-wide receipt and ships it to the connected printer.
6. **Settings** — edit Gemini key, kitchen name, disconnect / change printer.

## Architecture
- **Backend:** FastAPI (`/app/backend/server.py`). One business endpoint: `POST /api/generate-quote`. Uses `emergentintegrations.LlmChat` with `gemini-2.5-flash` and the user-supplied API key (key is never persisted server-side; it's only used per-request).
- **Storage:** MongoDB persists each generated quote in `quotes` collection for analytics/history (id, quote, tone, customer, order, kitchen, created_at — no PII keys).
- **Frontend:** Expo Router file-based routes — `index`, `setup`, `printer-setup`, `home`, `preview`, `settings`. Local state in AsyncStorage. Handwritten Caveat font loaded from Google Fonts CDN via `expo-font`.
- **Bluetooth printer:** `src/lib/printer.ts` lazy-loads `react-native-bluetooth-escpos-printer` when present (production APK builds) and gracefully falls back to a simulated scan + simulated print inside Expo Go so the entire UI flow is exercisable in the preview.

## Non-goals
- No multi-user / no auth (single-device tool).
- No order management — the order number is just a counter.

## Future ideas
- Tone preset (let user lock to warm/funny/motivational).
- Save customer history → personalize quotes ("Rahul, you crushed 12 orders!").
- Photo on the sticky note (chef selfie) printed via image ESC/POS commands.
